About OpenML

OpenML is a collection of tools that we use at Yamuna Software regularly that we have open-sourced to the public. We use these tools in our applications and we have others that use these libraries in their development environment.

The tools work through a genetic algorithm which takes in a series of inputs and evolves a single or a set of weights until a desired action is performed. Furthermore, a secondary algorithm is used as a lookup table to verify the results of the genetic algorithm. Scroll down to learn more.




JVision

This vision library is an add on library for basic vision tasks that can be done quickly and efficiently without having to load a very large library to run the code. This library can also be combined with another ML library to do more powerful vision tasks (such as image detection, feature recognition, etc) much faster than other libraries out there. The main advantage is the speed and usability across many devices for this library. This mainly fills the gap of having a small, yet powerful vision library, which can be used in an environment (such as Java) which is compatible with every device.

Some features that are included in this vision library are color detection, motion detection, image masks, image convolutions, distortion and noise correction, lighting analysis, low-level image processing, and real-life positioning capabilities using our VPS (Vision Positioning System) algorithm. This library and its features can be used alone for tasks such as basic image analysis, processing, or even AR. Additionally, the library can be augmented with a machine learning or neural network framework, to do some more complicated vision tasks, that use this vision library as a base.

Furthermore, if you would like to use our Surabhi.js machine learning framework to run this library in conjunction with, you can do so, using a ScriptEngine or running this code on a backend server and have the Surabhi.js library run in the frontend. Thus, if you want to, you can use both of our ML tools in your projects as well, which is very powerful as we have developed these internal tools with very complex projects in mind.

If you would like to use the JAmuna ML tool, listed below, in conjunction with this vision library, then it is very simple because both pieces of software are written in Java making it easy to integrate into any system, project, or environment. The best part about this integration is that Java is an Object-Oriented language which means that the user can use these classes easily as objects, which contain all of the methods and variables necessary to run Machine Learning pipelines.

 



JAmuna

The JAmuna Learning Library is similar to other Machine Learning libraries in that it uses supervised learning with traditional techniques such as weights, biases, and dot-products. However, the difference between most other learning libraries and JAmuna is that JAmuna uses a genetic algorithm to learn and evolve between an array of neural networks to attain the best possible network for the task at hand.

The JAmuna library is a machine learning library that is built from using neural networks to get better results from you testing. The library uses a special method to learn where it creates multiple neural networks and evolves to a better and better network with added data. The more that the data is aggregated and used to train this system, the better the resulting network will be. This method is a little more computationally expensive when compared to the traditional Back-Propagation Learning; however, the resulting training process is much easier to visualize and achieve great performance as well. The library is designed to handle classification and class-based machine learning tasks, which is perfect for many applications of the technology.

This is a special type of genetic algorithm that is based upon the evolution of asexually-reproducing bacteria. Additionally, this algorithm implements the Bottleneck Effect of Biology, and this effect is coupled with an asexual reproduction model (known as Antibiotic Resistance), to get the most accurate results quickly. Additionally, the algorithm takes the best child of each generation and reproduces that child node into a set number of children, using a weight mutation.
